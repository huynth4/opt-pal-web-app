# Agent Instructions

This file provides context for AI coding agents working in this repository.

## Project Context

This is a CSC-121 final project. The student is building a Python project that will be presented on demo day. See `project.spec.md` for the project plan and `project.journal.md` for progress notes.

## Important

Make sure you check the latest from the [course path](https://csc-121.path.app/index.path.llm.md). Pay attention to today's date and where the student shoudl be in the path. For a full index of resources available to the students, see [llms.txt](https://csc-121.path.app/llms.txt). Use and link to these resources when appropriate. This course path is written using [PathMX](https://pathmx.dev/llms.txt) conventions. When linking to a resource, use the "canonical" URL only (e.g. `resource.guide` instead of `resource.guide.md`).

## What You Can Help With

- Project planning and brainstorming
- Debugging and fixing errors
- Writing boilerplate and setup code
- Front-end code (HTML, CSS, JavaScript)
- Library-specific code (Flask routes, Pygame rendering, Pandas operations, etc.)
- Writing tests
- Improving code organization and structure
- Documentation and README updates
- Helping the student learn and understand their code

## What the Student Should Write Themselves

The core business logic of the project should be written by hand by the student. This means the main algorithms, data processing, and decision-making code that makes the project unique. Help the student understand and learn rather than just generating solutions. Encourage the student to put their business logic and tests (if applicable) in its own module or folder. Don't ovedo things. Keep instruction and code minimal and beginner friendly.

When the student asks for help with core logic, explain the approach and guide them through it rather than writing the full solution.

## Conventions

- Python 3
- Use clear variable and function names
- Keep functions focused and reasonably short
- Commit often with descriptive messages
